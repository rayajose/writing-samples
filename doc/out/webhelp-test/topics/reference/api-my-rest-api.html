My REST API

Reference documentation

Context

This API reference describes resources I created in a Spring Boot application. It is a simple REST API to use as subject matter for this sample document.

Note: The environment, code, and reference documentation are works in progress.

About the API

  • resource URIs and standard HTTP requests
  • simple operations using ideas found on the web
  • Java Spring Boot project on the backend
  • uses working Java code (For more information, see My Java class library.)
  • Postman used to test the requests and confirm the responses (For more information, see API testing with Postman.)

About the documentation

The document design is based on various public API References. While reviewing the public API references, the primary aspects considered were content and format.

Authentication

This service uses Spring Web Security API keys and secrets.

Base URI

My REST API application was installed and configured as a local host.

http://localhost:8080/api/

REST resource: xmen

The xmen resource allows you to manage records in the X-Men repository. It provides endpoints for performing HTTP CRUD operations on X-Men records.

Properties

These are the properties for the xmen REST resource.

Property Data Type Description
id int The unique identifier for the character.
name String The characterâs real name.
heroName String The characterâs hero name.
power String The characterâs powers.
debut String The comic book title and year of debut.
creator String The characterâs creators.

Endpoints

The xmen resource provides the following endpoints.

/ xmen

Creates a new X-Men entity.

Required parameters

Required parameters for the xmen POST request.

Header
Name Data Type Description
Content-Type String The media type of the resource sent in the HTTP message body. Use application/json.
API-KEY token The authentication key.
Body

The request body will contain a JSON object representing the new X-Men entity.

Name Data Type Description
id int The unique identifier for the character.
name String

The characterâs real name.

Format: <<firstName>> <<lastName>>
heroName String The characterâs hero name.
power String The characterâs powers.
debut String

The comic book title, issue number and year of debut.

Format: <<title>> <<issueNumber>> (<<year>>)
creator String The characterâs creators. Use commas to delimit multiple creators.
Example requests

Examples for the xmen POST requests.

    "heroName": "Dust",

})
headers = {
    'API-KEY': '{myApikey}',
    'Content-Type': 'application/json'
}
                    
response = requests.request("POST", url, headers=headers, data=payload)
                    
print(response.text)

REST resource: needles

The needles resource matches the specified words in a block of text. Send up to five different words (needles) and a single text block (haystack). Find the needles in the haystack.

Properties

Name Data Type Description
needles string array The set of words along with word count.
count int The number of matched words found.
needle string The matched word.

Endpoints

The needles resource provides the following endpoints.

/ needles

This endpoint is better suited for short blocks of text. Returns the match count for each specified word in a block of text. Up to five words can be specified in the request.

Required parameters
Query
Name Data Type Description
api_key token The authentication key.
haystack string The text block to search for word matches.
needles string array The words to match.

/ needles

This endpoint is better suited for long blocks of text. Returns the match count for each specified word in a block of text. Up to five words can be specified in the request.

Required parameters
Header
Name Data Type Description
Content-Type string Use application/json
API-KEY token The authentication key.
Body
Name Data Type Description
haystack string The text block to search for word matches.
needles string array The words to match.
Request

Examples for the needles POST request.

cURL
curl -L 'http://localhost:8080/api/needles' \
-H 'Content-Type: application/json' \
-H 'API-KEY: {myApikey}' \
-d '{
    "haystack":"Like finding a needle in a haystack",
    "needles":["finding","needle","a","haystack","needles"]
}'
NodeJs
var request = require('request');
var options = {
    'method': 'POST',
    'url': 'http://localhost:8080/api/needles',
    'headers': {
        'Content-Type': 'application/json',
        'API-KEY': '{myApikey}'
    },
    body: JSON.stringify({
        "haystack": "Like finding a needle in a haystack",
        "needles": [
            "finding",
            "needle",

};
request(options, function (error, response) {
    if (error) throw new Error(error);
    console.log(response.body);
});
Python
import requests
import json
            
url = "http://localhost:8080/api/needles"
            
payload = json.dumps({

If the request contains more than five needles, the response will return the following message:

"The maximum number of needles allowed is 5."

HTTP response status codes

200 OK

This is the standard response for a successful HTTP request. The actual message will depend on the specific request. Refer to the applicable resource section for specific 200 status code.

400 Bad Request

The request cannot be filled due to bad syntax.

404 Not Found

The requested resource could not be found but may be available again in the future. Subsequent requests by the client are permissible.

500 Internal Server Error

A generic error message given when no more specific message is suitable. For an authentication error, the application will return the following message: Missing API Key or Secret

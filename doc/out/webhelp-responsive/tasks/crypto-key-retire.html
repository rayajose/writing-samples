Retiring Keys

PCI-DSS-V4-3.7.5

  • Identify two key custodians.

  • Identify a member of the development team to help with issues.

  • Have following information on hand:

    • applicable Vault ID with token

    • applicable Environment ID with token

    • Encryption Tool custodian passphrase

    • current AD cert and credentials

  • Ensure access to the applicable environment.

The key pool in each environment contains 25 keys. Keys will be rotated 5 at a time. Unless overridden by manual selection, the Encryption Tool will automatically select the appropriate keys for the given operation.

Retire keys as follows:

  1. Open the Encryption Tool.
  2. Select the Key Operations tab.
    The Key Operations window will display.
  3. Select Retire from the Operation drop-down and populate the remaining fields.

    • Valut ID

    • Vault Token

    • Environment ID

    • Environment Token

    • Path to AD cert

    • Set Key Type to DEK

  4. To bypass automatic selection of keys by the Encryption Tool, click the Manually Pick Keys button. Select keys accordingly using the checkbox next to the key ID.
  5. Click the Submit button.
    Output window activity will begin. Monitor the output.

    • If key the operation fails, the output window will display failure messages. In this case, forward the log information to the developer on standby and close the Encryption Tool. The process will need to restart from step one once the issue has been resolved.

    • If the operation is successful, the output window will display the following message along with the list of new key IDs.

      Five keys <<operation>> successfully.

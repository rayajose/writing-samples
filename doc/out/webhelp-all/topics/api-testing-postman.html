API testing with Postman

How-to

This is a how-to document describing how I configured and tested the requests in My REST API using Postman.

Creating a Postman collection

Prerequisites:

  • Install Postman desktop application.

  • Sign in to Postman account.

  • Create a Postman workspace.

A collection in Postman is a set of API requests that can be configured for testing purposes. This task describes how to create a Postman collection using the REST API basics template.

To create a collection follow these steps:

  1. Select Collections in the sidebar, then select + > REST API basics.

    A new collection will appear with pre-built GET, POST, PUT, and DELETE request testers.

  2. Rename the collection using your REST API resource name. To rename your collection, open the drop-down menu and select Rename.

    This example uses the xmen resource from My REST API.

  3. Configure and run tests accordingly.

Testing the POST request

  • Common Prerequisites:

    • Install Postman desktop application.

    • Sign in to Postman account.

    • Create a Postman workspace.

    • Create a Postman collection using the REST API basics template.

This task describes how to test a POST request in Postman.

To test a POST request follow these steps:

  1. In the xmen collection, select POST Post data request type.

    The POST request builder will be displayed.

  2. Enter the base URL in the URL field: http://localhost:8080/api/xmen
  3. Use the URL field, the Params tab, the Headers tab, and the Body tab, as needed to set the request parameters.
  4. Click Send.
    If successful, the response will show the details for the X-men entity sent in the request.

POST parameters

Required parameters for the xmen POST request.

Header

Name Data Type Description
Content-Type String The media type of the resource sent in the HTTP message body. Use application/json.
API-KEY token The authentication key.

Body

The request body will contain a JSON object representing the new X-Men entity.

Name Data Type Description
id int The unique identifier for the character.
name String

The characterâs real name.

Format: <<firstName>> <<lastName>>
heroName String The characterâs hero name.
power String The characterâs powers.
debut String

The comic book title, issue number and year of debut.

Format: <<title>> <<issueNumber>> (<<year>>)
creator String The characterâs creators. Use commas to delimit multiple creators.

POST requests

Examples for the xmen POST requests.

cURL

curl -L 'http://localhost:8080/api/xmen' \
-H 'Content-Type: application/json' \
-X-API-KEY: '{myApikey}' 
-d '{
    "id": 4,
    "name": "Hisako Ichiki",
    "heroName": "Dust",
    "power": "Sand manipulation",
    "debut": "New X-Men #133 (2002)",
    "creator": "Grant Morrison, Ethan Van Sciver"
}'

NodeJs

var request = require('request');
var options = {
    'method': 'POST',
    'url': 'http://localhost:8080/api/xmen',
        'headers': {
            'API-KEY': '{myApikey}',
            'Content-Type': 'application/json'
        },
body: JSON.stringify({
    "id": 4,
    "name": "Hisako Ichiki",
    "heroName": "Dust",
    "power": "Sand manipulation",
    "debut": "New X-Men #133 (2002)",
    "creator": "Grant Morrison, Ethan Van Sciver"
    })                   
};
request(options, function (error, response) {
    if (error) throw new Error(error);
    console.log(response.body);
});

Python

import requests
import json
                    
url = "http://localhost:8080/api/xmen"
                    
payload = json.dumps({
    "id": 4,
    "name": "Hisako Ichiki",
    "heroName": "Dust",
    "power": "Sand manipulation",
    "debut": "New X-Men #133 (2002)",
    "creator": "Grant Morrison, Ethan Van Sciver"
})
headers = {
    'API-KEY': '{myApikey}',
    'Content-Type': 'application/json'
}
                    
response = requests.request("POST", url, headers=headers, data=payload)
                    
print(response.text)

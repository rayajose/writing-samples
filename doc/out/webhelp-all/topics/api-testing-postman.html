API testing with Postman

This is a how-to document describing how I configured and tested the requests in My REST API using Postman.

Creating a Postman collection

Prerequisites:

  • Install Postman desktop application.

  • Sign in to Postman account.

  • Create a Postman workspace.

A collection in Postman is a set of API requests that can be configured for testing purposes. This task describes how to create a Postman collection using the REST API basics template.

To create a collection follow these steps:

  1. Select Collections in the sidebar, then select + > REST API basics.

    A new collection will appear with pre-built GET, POST, PUT, and DELETE request testers.

  2. Rename the collection using your REST API resource name. To rename your collection, open the drop-down menu and select Rename.

    This example uses the xmen resource from My REST API.

  3. Configure and run tests accordingly.

Testing the POST request

  • Common Prerequisites:

    • Install Postman desktop application.

    • Sign in to Postman account.

    • Create a Postman workspace.

    • Create a Postman collection using the REST API basics template.

This task describes how to test a POST request in Postman.

To test a POST request follow these steps:

  1. In the xmen collection, select POST Post data request type.

    The POST request builder will be displayed.

  2. Enter the base URL in the URL field: http://localhost:8080/api/xmen
  3. Use the URL field, the Params tab, the Headers tab, and the Body tab, as needed to set the request parameters.
  4. Click Send.
    If successful, the response will show the details for the X-men entity sent in the request.

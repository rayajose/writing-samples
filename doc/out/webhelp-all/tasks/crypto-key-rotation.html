Key Rotation

PCI-DSS-V4-3.7.4, PCI-DSS-V4-3.7.5

This task describes the process for rotating cryptographic keys to ensure security and compliance with Company policies.

Important: Rotation requires split knowledge for creating new keys. The process will require 2 members of the key_custodian security group.

Keys are generated with a maximum lifetime of 5 years. However, 5 of the 25 are replaced every six months making their active lifespan roughly 2.5 years. Keys are rotated as follows:

  1. Identify the 5 oldest keys in the set for rotation.
  2. Using the Encryption Tool, create 5 new keys.
  3. Update systems to use new keys.
  4. Replace old keys with the newly generated keys in all systems that use them.

    Update key configurations in:

    • Encryption libraries
    • Database encryption settings
    • SSL/TLS certificate stores
    • API gateways
  5. Re-key the data encrypted by the disabled keys.
  6. Ensure that encryption, decryption, and secure communication is as expected with the new keys.
  7. Disable the oldest 5 keys.
  8. Maintain disabled keys in a decrypt-only state for three and half years.
  9. Remove the oldest 5 keys from the pool.
  10. Update the key management record on the Company wiki. For each key created and disabled, the record should include the following:
    • ID
    • name of the database where it is used
    • names of custodians who created them
    • date of rotation
  11. Notify Development Lead of the key update.
